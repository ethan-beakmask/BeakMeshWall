Threat Feed API 整合說明

BeakMeshWall 提供 REST API 供外部系統（如 SIEM、WAF、登入監控平台）自動推送 IP 封鎖/解除請求。

1. 建立信任 -- API Key

外部系統需要一把 API Key 才能呼叫 BeakMeshWall API。API Key 由管理員在 Central Server 上透過 CLI 產生。

產生 API Key
# 產生 threat scope 的 key（只能 block / unblock / 讀取白名單）
flask create-api-key --name "beakplatform-prod" --scope threat

# 產生 full scope 的 key（可修改白名單）
flask create-api-key --name "soc-admin" --scope full

產生後會顯示完整的 key，只會顯示一次，請妥善保存。

API Key Scope（權限範圍）
Scopeblock / unblock讀取白名單修改白名單
threat可以可以不可以
full可以可以可以
管理 API Key
# 列出所有 API Key
flask list-api-keys
2. 傳送封鎖 / 解除指令

所有 API 請求需在 HTTP Header 帶入 X-API-Key。

封鎖 IP（Block）
curl -X POST https://<central>/api/v1/threat/block \
  -H "Content-Type: application/json" \
  -H "X-API-Key: <your_api_key>" \
  -d '{
    "source": "beakplatform",
    "ip": "203.0.113.50",
    "reason": "brute_force",
    "detail": "login_fail_10_in_5m",
    "duration": 3600
  }'
欄位必填說明
ip是要封鎖的 IP 或 CIDR（支援 IPv4/IPv6）
source否來源系統名稱，預設 "external"
reason否封鎖原因（如 brute_force、port_scan）
detail否詳細說明
duration否封鎖秒數，省略則永久封鎖
解除封鎖（Unblock）
curl -X POST https://<central>/api/v1/threat/unblock \
  -H "Content-Type: application/json" \
  -H "X-API-Key: <your_api_key>" \
  -d '{"ip": "203.0.113.50"}'
查詢目前封鎖清單
curl https://<central>/api/v1/threat/blocks \
  -H "X-API-Key: <your_api_key>"
3. 白名單機制

白名單優先原則：白名單內的 IP/CIDR 無法被封鎖。任何 block 請求命中白名單時，會被拒絕並回傳 HTTP 409。

白名單的增刪需要 full scope 的 API Key，或由管理員透過 Web UI / CLI 操作。

CLI 管理白名單
# 新增
flask whitelist-add --ip "192.168.0.0/24" --description "Internal network"

# 列出
flask whitelist-list

# 移除
flask whitelist-remove --id 1
API 管理白名單（需 full scope）
# 列出
curl https://<central>/api/v1/threat/whitelist \
  -H "X-API-Key: <your_api_key>"

# 新增
curl -X POST https://<central>/api/v1/threat/whitelist \
  -H "Content-Type: application/json" \
  -H "X-API-Key: <full_scope_key>" \
  -d '{"ip_cidr": "10.0.0.0/8", "description": "Private network"}'

# 移除
curl -X DELETE https://<central>/api/v1/threat/whitelist/<id> \
  -H "X-API-Key: <full_scope_key>"
4. 回應代碼
HTTP Codestatus說明
201accepted封鎖成功，任務已派發到節點
200accepted解除封鎖成功
409rejected命中白名單，封鎖被拒絕
409already_blockedIP 已在封鎖中
401-API Key 缺失或無效
403-API Key scope 不足
5. EDL 匯出（硬體防火牆整合）

每次黑白名單變動時，BeakMeshWall 會自動匯出純文字檔：

檔案內容用途
blocklist.txt目前活躍的封鎖 IP，一行一個Palo Alto EDL / Fortinet External Connector
whitelist.txt白名單 IP/CIDR，一行一個防火牆允許清單

匯出路徑可透過環境變數 BMW_EDL_EXPORT_DIR 設定。

Threat Feed API Integration Guide

BeakMeshWall provides a REST API for external systems (SIEM, WAF, login monitoring platforms) to automatically push IP block/unblock requests.

1. Establishing Trust -- API Key

External systems need an API Key to call BeakMeshWall APIs. Keys are generated by an administrator on the Central Server via CLI.

Generate an API Key
# Generate a threat-scope key (block / unblock / read whitelist only)
flask create-api-key --name "beakplatform-prod" --scope threat

# Generate a full-scope key (can also modify whitelist)
flask create-api-key --name "soc-admin" --scope full

The full key is displayed only once upon creation. Store it securely.

API Key Scopes
Scopeblock / unblockRead whitelistModify whitelist
threatYesYesNo
fullYesYesYes
Manage API Keys
# List all API keys
flask list-api-keys
2. Sending Block / Unblock Commands

All API requests require the X-API-Key HTTP header.

Block an IP
curl -X POST https://<central>/api/v1/threat/block \
  -H "Content-Type: application/json" \
  -H "X-API-Key: <your_api_key>" \
  -d '{
    "source": "beakplatform",
    "ip": "203.0.113.50",
    "reason": "brute_force",
    "detail": "login_fail_10_in_5m",
    "duration": 3600
  }'
FieldRequiredDescription
ipYesIP address or CIDR to block (IPv4/IPv6)
sourceNoSource system name, defaults to "external"
reasonNoBlock reason (e.g. brute_force, port_scan)
detailNoDetailed description
durationNoBlock duration in seconds. Omit for permanent.
Unblock an IP
curl -X POST https://<central>/api/v1/threat/unblock \
  -H "Content-Type: application/json" \
  -H "X-API-Key: <your_api_key>" \
  -d '{"ip": "203.0.113.50"}'
Query Active Blocks
curl https://<central>/api/v1/threat/blocks \
  -H "X-API-Key: <your_api_key>"
3. Whitelist Mechanism

Whitelist-first principle: IPs/CIDRs on the whitelist can never be blocked. Any block request matching a whitelist entry is rejected with HTTP 409.

Whitelist modifications require a full scope API Key, or can be done by an administrator via Web UI / CLI.

CLI Whitelist Management
# Add
flask whitelist-add --ip "192.168.0.0/24" --description "Internal network"

# List
flask whitelist-list

# Remove
flask whitelist-remove --id 1
API Whitelist Management (requires full scope)
# List
curl https://<central>/api/v1/threat/whitelist \
  -H "X-API-Key: <your_api_key>"

# Add
curl -X POST https://<central>/api/v1/threat/whitelist \
  -H "Content-Type: application/json" \
  -H "X-API-Key: <full_scope_key>" \
  -d '{"ip_cidr": "10.0.0.0/8", "description": "Private network"}'

# Remove
curl -X DELETE https://<central>/api/v1/threat/whitelist/<id> \
  -H "X-API-Key: <full_scope_key>"
4. Response Codes
HTTP CodestatusDescription
201acceptedBlock accepted, tasks dispatched to nodes
200acceptedUnblock successful
409rejectedWhitelist hit, block refused
409already_blockedIP is already actively blocked
401-Missing or invalid API Key
403-Insufficient API Key scope
5. EDL Export (Hardware Firewall Integration)

BeakMeshWall automatically exports plain-text files whenever the blocklist or whitelist changes:

FileContentUse Case
blocklist.txtCurrently active blocked IPs, one per linePalo Alto EDL / Fortinet External Connector
whitelist.txtWhitelisted IPs/CIDRs, one per lineFirewall allow lists

Export path is configurable via the BMW_EDL_EXPORT_DIR environment variable.
