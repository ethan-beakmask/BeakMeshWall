# Installation Guide

`install.sh` is a single-file installer/upgrader for both **Central Server** and **Agent**. It is designed to be run on Ubuntu 22.04 / 24.04 LTS and pulls source from GitHub.

```
sudo bash install.sh                            # show help
sudo bash install.sh central install            # install Central
sudo bash install.sh central upgrade            # upgrade Central
sudo bash install.sh central status             # show Central status
sudo bash install.sh central uninstall          # remove Central
sudo bash install.sh agent install              # install Agent
sudo bash install.sh agent upgrade              # upgrade Agent
sudo bash install.sh agent status               # show Agent status
sudo bash install.sh agent uninstall            # remove Agent
```

The script is idempotent: running `install` on top of an existing deployment offers to switch into upgrade mode automatically.

---

## Central Server

### What gets installed

| Component | Default location |
|-----------|------------------|
| App code (git checkout) | `/opt/beakmeshwall/` |
| Python venv | `/opt/beakmeshwall/venv/` |
| Environment file | `/opt/beakmeshwall/.env` |
| systemd service | `beakmeshwall-central.service` (gunicorn, port 5102 internal) |
| Nginx vhost | `/etc/nginx/sites-enabled/beakmeshwall-central` (port 5101 external) |
| PostgreSQL DB | `beakmeshwall` (user: `beakmeshwall`, password autogenerated) |
| Service user | `beakmeshwall` (system, nologin) |

### Quick install

```bash
curl -sLO https://raw.githubusercontent.com/ethan-beakmask/BeakMeshWall/main/install.sh
sudo bash install.sh central install
```

The installer will prompt for the initial admin password (set `ADMIN_PASSWORD=...` to skip).

### Environment variables (Central)

| Variable | Default | Purpose |
|---|---|---|
| `CENTRAL_DIR` | `/opt/beakmeshwall` | Install root |
| `CENTRAL_PORT` | `5101` | External Nginx port (auto-finds free port if taken) |
| `DB_NAME` | `beakmeshwall` | PostgreSQL database |
| `DB_USER` | `beakmeshwall` | DB user |
| `DB_PASS` | *autogenerated* | DB password (32 hex chars) |
| `ADMIN_USERNAME` | `admin` | Initial admin user |
| `ADMIN_PASSWORD` | *prompted* | Initial admin password (>= 8 chars) |
| `GITHUB_REPO` | `https://github.com/ethan-beakmask/BeakMeshWall.git` | Source repo |
| `GIT_BRANCH` | `main` | Branch to track |
| `GITHUB_TOKEN` | *empty* | PAT for private repos |

### Upgrade

```bash
sudo bash /opt/beakmeshwall/install.sh central upgrade
```

Upgrade preserves: `.env`, DB data, Nginx vhost, generated certificates. It only updates source, Python deps, runs `flask db upgrade`, and rewrites the systemd unit (port unchanged).

If you need to change the port or rebuild Nginx vhost, run `central uninstall` then `central install`.

## Agent

### What gets installed

| Component | Default location |
|---|---|
| Source checkout (for `go build`) | `/opt/beakmeshwall-agent/` |
| Compiled binary | `/usr/local/bin/bmw-agent` |
| Configuration | `/etc/beakmeshwall/agent.yaml` |
| systemd service | `beakmeshwall-agent.service` (runs as root, needed for nftables) |

### Prerequisites

- Go 1.22+ (`apt-get install golang-go` on Ubuntu 24.04)
- `nftables` (`apt-get install nftables`)

The installer ensures these are present.

### Quick install

```bash
curl -sLO https://raw.githubusercontent.com/ethan-beakmask/BeakMeshWall/main/install.sh
CENTRAL_URL=http://192.168.0.16:5101 sudo bash install.sh agent install
```

The agent self-registers with Central on first install and stores the bearer token in `/etc/beakmeshwall/agent.yaml`. On subsequent upgrades the existing token is reused.

### Environment variables (Agent)

| Variable | Default | Purpose |
|---|---|---|
| `AGENT_DIR` | `/opt/beakmeshwall-agent` | Source checkout location |
| `CENTRAL_URL` | *required* | Central URL the agent reports to |
| `AGENT_HOSTNAME` | `$(hostname)` | Override hostname reported to Central |
| `AGENT_TOKEN` | *empty* | Pre-existing token (skips registration) |

### Upgrade

```bash
sudo bash /opt/beakmeshwall-agent/install.sh agent upgrade
```

Upgrade preserves: `agent.yaml` (including token), nftables `inet beakmeshwall` table state. It re-pulls source, rebuilds the binary, and restarts the service.

---

## Self-managed deployment (Central + Agent on the same host)

Running an agent on the Central host is supported and is a reasonable choice when you want Central to manage its own firewall:

```bash
sudo bash install.sh central install
CENTRAL_URL=http://127.0.0.1:5101 sudo bash install.sh agent install
```

The two components use independent service users (`beakmeshwall` for Central, `root` for Agent) and do not conflict.

---

## Uninstall notes

`central uninstall` drops the PostgreSQL database and removes the service user.
`agent uninstall` removes the systemd unit, binary, config, and source checkout, but **does not** flush the `inet beakmeshwall` nftables table -- run `nft delete table inet beakmeshwall` manually if desired.
